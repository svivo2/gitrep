Get-Command

\r\n\r\n
\r\n\t\t
\r\n\t\t\t
Module:
\r\n\t\t\t
Microsoft.PowerShell.Core
\r\n\t\t
\r\n
\r\n\r\n\t
\r\n\t\t
\r\n\t\t\t

Gets all commands.

\n\r\n\t\t
\r\n\t
\r\n\r\n
\r\n

In this article

\r\n
\r\n\t

Syntax

\r\n\t
Get-Command\n   [-Verb <String[]>]\n   [-Noun <String[]>]\n   [-Module <String[]>]\n   [-FullyQualifiedModule <ModuleSpecification[]>]\n   [-TotalCount <Int32>]\n   [-Syntax]\n   [-ShowCommandInfo]\n   [[-ArgumentList] <Object[]>]\n   [-All]\n   [-ListImported]\n   [-ParameterName <String[]>]\n   [-ParameterType <PSTypeName[]>]\n   [<CommonParameters>]
\r\n\t
Get-Command\n   [[-Name] <String[]>]\n   [-Module <String[]>]\n   [-FullyQualifiedModule <ModuleSpecification[]>]\n   [-CommandType <CommandTypes>]\n   [-TotalCount <Int32>]\n   [-Syntax]\n   [-ShowCommandInfo]\n   [[-ArgumentList] <Object[]>]\n   [-All]\n   [-ListImported]\n   [-ParameterName <String[]>]\n   [-ParameterType <PSTypeName[]>]\n   [-UseFuzzyMatching]\n   [-UseAbbreviationExpansion]\n   [<CommonParameters>]
\r\n\r\n\t

Description

\r\n\t

The Get-Command cmdlet gets all commands that are installed on the computer, including cmdlets,\naliases, functions, filters, scripts, and applications. Get-Command gets the commands from\nPowerShell modules and commands that were imported from other sessions. To get only commands that\nhave been imported into the current session, use the ListImported parameter.

\n

Without parameters, Get-Command gets all of the cmdlets, functions, and aliases installed on the\ncomputer. Get-Command * gets all types of commands, including all of the non-PowerShell files in\nthe Path environment variable ($env:Path), which it lists in the Application command type.

\n

Get-Command that uses the exact name of the command, without wildcard characters, automatically\nimports the module that contains the command so that you can use the command immediately. To enable,\ndisable, and configure automatic importing of modules, use the $PSModuleAutoLoadingPreference\npreference variable. For more information, see about_Preference_Variables.

\n

Get-Command gets its data directly from the command code, unlike Get-Help, which gets its\ninformation from help topics.

\n

Starting in Windows PowerShell 5.0, results of the Get-Command cmdlet display a Version column\nby default. A new Version property has been added to the CommandInfo class.

\n\r\n\r\n\t

Examples

\r\n\t

Example 1: Get cmdlets, functions, and aliases

\r\n\t

This command gets the PowerShell cmdlets, functions, and aliases that are installed on the computer.

\n\r\n\t
Get-Command
\r\n\t\r\n\t

Example 2: Get commands in the current session

\r\n\t

This command uses the ListImported parameter to get only the commands in the current session.

\n\r\n\t
Get-Command -ListImported
\r\n\t\r\n\t

Example 3: Get cmdlets and display them in order

\r\n\t

This command gets all of the cmdlets, sorts them alphabetically by the noun in the cmdlet name, and\nthen displays them in noun-based groups. This display can help you find the cmdlets for a task.

\n\r\n\t
Get-Command -Type Cmdlet | Sort-Object -Property Noun | Format-Table -GroupBy Noun
\r\n\t\r\n\t

Example 4: Get commands in a module

\r\n\t

This command uses the Module parameter to get the commands in the Microsoft.PowerShell.Security\nand Microsoft.PowerShell.Utility modules.

\n\r\n\t
Get-Command -Module Microsoft.PowerShell.Security, Microsoft.PowerShell.Utility
\r\n\t\r\n\t

Example 5: Get information about a cmdlet

\r\n\t

This command gets information about the Get-AppLockerPolicy cmdlet. It also imports the\nAppLocker module, which adds all of the commands in the AppLocker module to the current\nsession.

\n\r\n\t
Get-Command Get-AppLockerPolicy
\r\n\t

When a module is imported automatically, the effect is the same as using the Import-Module cmdlet.\nThe module can add commands, types and formatting files, and run scripts in the session. To enable,\ndisable, and configuration automatic importing of modules, use the $PSModuleAutoLoadingPreference\npreference variable. For more information, see about_Preference_Variables.

\n\r\n\t

Example 6: Get the syntax of a cmdlet

\r\n\t

This command uses the ArgumentList and Syntax parameters to get the syntax of the Get-ChildItem\ncmdlet when it is used in the Cert: drive. The Cert: drive is a PowerShell drive that the\nCertificate Provider adds to the session.

\n\r\n\t
Get-Command Get-Childitem -Args Cert: -Syntax
\r\n\t

When you compare the syntax displayed in the output with the syntax that is displayed when you omit\nthe Args (ArgumentList) parameter, you\'ll see that the Certificate provider adds a dynamic\nparameter, CodeSigningCert, to the Get-ChildItem cmdlet.

\n

For more information about the Certificate provider, see about_Certificate_Provider.

\n\r\n\t

Example 7: Get dynamic parameters

\r\n\t

The command in the example uses the Get-DynamicParameters function to get the dynamic parameters\nthat the Certificate provider adds to the Get-ChildItem cmdlet when it is used in the Cert: drive.

\n\r\n\t
function Get-DynamicParameters\n{\n    param ($Cmdlet, $PSDrive)\n    (Get-Command $Cmdlet -ArgumentList $PSDrive).ParameterSets | ForEach-Object {$_.Parameters} | Where-Object { $_.IsDynamic } | Select-Object -Property Name -Unique\n}\nGet-DynamicParameters -Cmdlet Get-ChildItem -PSDrive Cert:\n\nName\n----\nCodeSigningCert
\r\n\t

The Get-DynamicParameters function in this example gets the dynamic parameters of a cmdlet. This\nis an alternative to the method used in the previous example. Dynamic parameter can be added to a\ncmdlet by another cmdlet or a provider.

\n\r\n\t

Example 8: Get all commands of all types

\r\n\t

This command gets all commands of all types on the local computer, including executable files in the\npaths of the Path environment variable ($env:path).

\n\r\n\t
Get-Command *
\r\n\t

It returns an ApplicationInfo object (System.Management.Automation.ApplicationInfo) for each\nfile, not a FileInfo object (System.IO.FileInfo).

\n\r\n\t

Example 9: Get cmdlets by using a parameter name and type

\r\n\t

This command gets cmdlets that have a parameter whose name includes Auth and whose type is\nAuthenticationMechanism.

\n\r\n\t
Get-Command -ParameterName *Auth* -ParameterType AuthenticationMechanism
\r\n\t

You can use a command like this one to find cmdlets that let you specify the method that is used to\nauthenticate the user.

\n

The ParameterType parameter distinguishes parameters that take an AuthenticationMechanism\nvalue from those that take an AuthenticationLevel parameter, even when they have similar names.

\n\r\n\t

Example 10: Get an alias

\r\n\t

This example shows how to use the Get-Command cmdlet with an alias.

\n\r\n\t
Get-Command dir\n\nCommandType     Name                                               ModuleName\n-----------     ----                                               ----------\nAlias           dir -> Get-ChildItem
\r\n\t

Although it is typically used on cmdlets and functions, Get-Command also gets scripts, functions,\naliases, and executable files.

\n

The output of the command shows the special view of the Name property value for aliases. The\nview shows the alias and the full command name.

\n\r\n\t

Example 11: Get all instances of the Notepad command

\r\n\t

This example uses the All parameter of the Get-Command cmdlet to show all instances of the "Notepad" command on the local computer.

\n\r\n\t
Get-Command Notepad -All | Format-Table CommandType, Name, Definition\n\nCommandType     Name           Definition\n-----------     ----           ----------\nApplication     notepad.exe    C:\\WINDOWS\\system32\\notepad.exe\nApplication     NOTEPAD.EXE    C:\\WINDOWS\\NOTEPAD.EXE
\r\n\t

The All parameter is useful when there is more than one command with the same name in the\nsession.

\n

Beginning in Windows PowerShell 3.0, by default, when the session includes multiple commands with\nthe same name, Get-Command gets only the command that runs when you type the command name. With\nthe All parameter, Get-Command gets all commands with the specified name and returns them in\nexecution precedence order. To run a command other than the first one in the list, type the fully\nqualified path to the command.

\n

For more information about command precedence, see about_Command_Precedence.

\n\r\n\t

Example 12: Get the name of a module that contains a cmdlet

\r\n\t

This command gets the name of the module in which the Get-Date cmdlet originated.\nThe command uses the ModuleName property of all commands.

\n\r\n\t
(Get-Command Get-Date).ModuleName\n\nMicrosoft.PowerShell.Utility
\r\n\t

This command format works on commands in PowerShell modules, even if they are not imported into the session.

\n\r\n\t

Example 13: Get cmdlets and functions that have an output type

\r\n\t\r\n\t
Get-Command -Type Cmdlet | Where-Object OutputType | Format-List -Property Name, OutputType
\r\n\t

This command gets the cmdlets and functions that have an output type and the type of objects that they return.

\n

The first part of the command gets all cmdlets.\nA pipeline operator (|) sends the cmdlets to the Where-Object cmdlet, which selects only the ones in which the OutputType property is populated.\nAnother pipeline operator sends the selected cmdlet objects to the Format-List cmdlet, which displays the name and output type of each cmdlet in a list.

\n

The OutputType property of a CommandInfo object has a non-null value only when the cmdlet code defines the OutputType attribute for the cmdlet.

\n\r\n\t

Example 14: Get cmdlets that take a specific object type as input

\r\n\t\r\n\t
Get-Command -ParameterType (((Get-NetAdapter)[0]).PSTypeNames)\n\nCommandType     Name                                               ModuleName\n-----------     ----                                               ----------\nFunction        Disable-NetAdapter                                 NetAdapter\nFunction        Enable-NetAdapter                                  NetAdapter\nFunction        Rename-NetAdapter                                  NetAdapter\nFunction        Restart-NetAdapter                                 NetAdapter\nFunction        Set-NetAdapter                                     NetAdapter
\r\n\t

This command finds cmdlets that take net adapter objects as input. You can use this command format\nto find the cmdlets that accept the type of objects that any command returns.

\n

The command uses the PSTypeNames intrinsic property of all objects, which gets the types that\ndescribe the object. To get the PSTypeNames property of a net adapter, and not the\nPSTypeNames property of a collection of net adapters, the command uses array notation to get the\nfirst net adapter that the cmdlet returns.

\n\r\n\t

Example 15: Get commands using a fuzzy match

\r\n\t

In this example, the name of the command deliberately has a typo as \'get-commnd\'.\nUsing the -UseFuzzyMatching switch, the cmdlet determined that the best match\nwas Get-Command followed by other native commands on the system that were a\nsimilar match.

\n\r\n\t
Get-Command get-commnd -UseFuzzyMatching\n\nCommandType     Name                                               Version    Source\n-----------     ----                                               -------    ------\nCmdlet          Get-Command                                        6.2.0.0    Microsoft.PowerShell.Core\nApplication     getconf                                            0.0.0.0    /usr/bin/getconf\nApplication     command                                            0.0.0.0    /usr/bin/command
\r\n\t\r\n\r\n\t

Parameters

\r\n\t
-All
\r\n\t
\r\n\t\t

Indicates that this cmdlet gets all commands, including commands of the same type that have the same\nname. By default, Get-Command gets only the commands that run when you type the command name.

\n

For more information about the method that PowerShell uses to select the command to run when\nmultiple commands have the same name, see about_Command_Precedence.\nFor information about module-qualified command names and running commands that do not run by default\nbecause of a name conflict, see about_Modules.

\n

This parameter was introduced in Windows PowerShell 3.0.

\n

In Windows PowerShell 2.0, Get-Command gets all commands by default.

\n\r\n\t\r\n\t\t\r\n\t\t\t\t\r\n\t\t\t\t\r\n\t\t\t\t\r\n\t\t\t\r\n\t\t\t\r\n\t\t
Type:SwitchParameter
Position:Named
Default value:None
Accept pipeline input:True
Accept wildcard characters:False
\r\n\t
\t
-ArgumentList
\r\n\t
\r\n\t\t

Specifies an array of arguments. This cmdlet gets information about a cmdlet or function when it is\nused with the specified parameters ("arguments"). The alias for ArgumentList is Args.

\n

To detect dynamic parameters that are available only when certain other parameters are used, set the\nvalue of ArgumentList to the parameters that trigger the dynamic parameters.

\n

To detect the dynamic parameters that a provider adds to a cmdlet, set the value of the\nArgumentList parameter to a path in the provider drive, such as WSMan:, HKLM:, or Cert:. When\nthe command is a PowerShell provider cmdlet, enter only one path in each command. The provider\ncmdlets return only the dynamic parameters for the first path the value of ArgumentList. For\ninformation about the provider cmdlets, see about_Providers.

\n\r\n\t\r\n\t\t\r\n\t\t\t\t\r\n\t\t\t\t\r\n\t\t\t\t\r\n\t\t\t\t\r\n\t\t\t\r\n\t\t\t\r\n\t\t
Type:Object[]
Aliases:Args
Position:1
Default value:None
Accept pipeline input:True
Accept wildcard characters:False
\r\n\t
\t
-CommandType
\r\n\t
\r\n\t\t

Specifies the types of commands that this cmdlet gets. Enter one or more command types. Use\nCommandType or its alias, Type. By default, Get-Command gets all cmdlets, functions, and\naliases.

\n

The acceptable values for this parameter are:

\n
    \n
  • Alias. Gets the aliases of all PowerShell commands. For more information, see about_Aliases.
    • \n
  • All. Gets all command types. This parameter value is the equivalent of Get-Command *.
    • \n
  • Application. Gets non-PowerShell files in paths listed in the Path environment\nvariable ($env:path), including .txt, .exe, and .dll files. For more information about the\nPath environment variable, see about_Environment_Variables.
    • \n
  • Cmdlet. Gets all cmdlets.
    • \n
  • ExternalScript. Gets all .ps1 files in the paths listed in the Path environment variable\n($env:path).
    • \n
  • Filter and Function. Gets all PowerShell advanced and simple functions and filters.
    • \n
  • Script. Gets all script blocks. To get PowerShell scripts (.ps1 files), use the ExternalScript\nvalue.
    • \n
\n\r\n\t\r\n\t\t\r\n\t\t\t\t\r\n\t\t\t\t\r\n\t\t\t\t\r\n\t\t\t\t\r\n\t\t\t\t\r\n\t\t\t\r\n\t\t\t\r\n\t\t
Type:CommandTypes
Aliases:Type
Accepted values:Alias, Function, Filter, Cmdlet, ExternalScript, Application, Script, Workflow, Configuration, All
Position:Named
Default value:None
Accept pipeline input:True
Accept wildcard characters:False
\r\n\t
\t
-FullyQualifiedModule
\r\n\t
\r\n\t\t

Specifies modules with names that are specified in the form of ModuleSpecification objects,\ndescribed in the Remarks section of\nModuleSpecification Constructor (Hashtable).\nFor example, the FullyQualifiedModule parameter accepts a module name that is specified in one\nof the following formats:

\n
    \n
  • @{ModuleName = "modulename"; ModuleVersion = "version_number"}
    • \n
  • @{ModuleName = "modulename"; ModuleVersion = "version_number"; Guid = "GUID"}
    • \n
\n

ModuleName and ModuleVersion are required, but Guid is optional.

\n

You cannot specify the FullyQualifiedModule parameter in the same command as a Module\nparameter. The two parameters are mutually exclusive.

\n\r\n\t\r\n\t\t\r\n\t\t\t\t\r\n\t\t\t\t\r\n\t\t\t\t\r\n\t\t\t\r\n\t\t\t\r\n\t\t
Type:ModuleSpecification[]
Position:Named
Default value:None
Accept pipeline input:True
Accept wildcard characters:False
\r\n\t
\t
-ListImported
\r\n\t
\r\n\t\t

Indicates that this cmdlet gets only commands in the current session.

\n

Starting in PowerShell 3.0, by default, Get-Command gets all installed commands,\nincluding, but not limited to, the commands in the current session. In PowerShell 2.0, it\ngets only commands in the current session.

\n

This parameter was introduced in Windows PowerShell 3.0.

\n\r\n\t\r\n\t\t\r\n\t\t\t\t\r\n\t\t\t\t\r\n\t\t\t\t\r\n\t\t\t\r\n\t\t\t\r\n\t\t
Type:SwitchParameter
Position:Named
Default value:None
Accept pipeline input:True
Accept wildcard characters:False
\r\n\t
\t
-Module
\r\n\t
\r\n\t\t

Specifies an array of modules. This cmdlet gets the commands that came from the specified modules.\nEnter the names of modules or module objects.

\n

This parameter takes string values, but the value of this parameter can also be a PSModuleInfo\nobject, such as the objects that the Get-Module and Import-PSSession cmdlets return.

\n\r\n\t\r\n\t\t\r\n\t\t\t\t\r\n\t\t\t\t\r\n\t\t\t\t\r\n\t\t\t\t\r\n\t\t\t\r\n\t\t\t\r\n\t\t
Type:String[]
Aliases:PSSnapin
Position:Named
Default value:None
Accept pipeline input:True
Accept wildcard characters:True
\r\n\t
\t
-Name
\r\n\t
\r\n\t\t

Specifies an array of names. This cmdlet gets only commands that have the specified name. Enter a\nname or name pattern. Wildcard characters are permitted.

\n

To get commands that have the same name, use the All parameter. When two commands have the same\nname, by default, Get-Command gets the command that runs when you type the command name.

\n\r\n\t\r\n\t\t\r\n\t\t\t\t\r\n\t\t\t\t\r\n\t\t\t\t\r\n\t\t\t\r\n\t\t\t\r\n\t\t
Type:String[]
Position:0

Notes

\r\n\t
    \n
  • When more than one command that has the same name is available to the session, Get-Command\nreturns the command that runs when you type the command name. To get commands that have the same\nname, listed in run order, use the All parameter. For more information, see\nabout_Command_Precedence.
    • \n
  • When a module is imported automatically, the effect is the same as using the Import-Module\ncmdlet. The module can add commands, types and formatting files, and run scripts in the session.\nTo enable, disable, and configuration automatic importing of modules, use the\n$PSModuleAutoLoadingPreference preference variable. For more information, see\nabout_Preference_Variables.
